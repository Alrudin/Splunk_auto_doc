# Domain Glossary

* **Edge List**: A flat list of single-hop `sourceNode -> targetNode` relationships extracted from Splunk data. Instead of computing full multi-hop lineage paths (e.g., UF -> HF -> IDX) in the Splunk backend, the backend provides the raw edge list, and the frontend (D3.js) is responsible for traversing and stitching the full directed graph natively.
* **Loss Ratio**: A metric designed to detect "vanishing" data by comparing ingress volume versus egress volume for a given node. 
  * *Note: Currently mocked in the data engine (`savedsearches.conf`) using a hardcoded low-EPS threshold to unblock UI development. Must be revisited in Task 1.2 to implement true ingress vs. egress delta calculations.*
* **Intelligent Aggregation**: The dynamic grouping of low-volume nodes (typically Universal Forwarders) in the UI to prevent graph clutter. When a tier has more than 10 nodes, the UI displays the top 10 by volume and collapses the rest into an expandable "Other" group. This logic is executed entirely on the client-side (D3.js) using the raw Edge List, preserving the ability to drill down without querying the backend again.
* **Focus Mode**: A UI filtering state triggered by selecting a specific `sourcetype`. To aggressively declutter the canvas and highlight the lineage path, any nodes and edges that do *not* handle the selected sourcetype are completely removed from the graph rendering until the filter is cleared.
* **Inspector Panel**: A detail view that appears when a user clicks a node or edge. To ensure a snappy, low-latency UI, it only displays in-memory data already retrieved via the Edge List (e.g., EPS, Loss Ratio) rather than executing ad-hoc Splunk searches. For deeper investigations, the panel will provide an "Open in Search" link.
